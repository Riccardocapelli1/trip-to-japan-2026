# Spec: Destination Discovery & Alternatives

## Goal
Provide a way to explore destinations beyond the current itinerary, offering suggestions and alternatives based on user interests or potential plan changes.

## User Persona
- **The Spontaneous Planner**: Wants to see what else Japan has to offer.
- **The Adaptive Traveler**: Needs a backup plan if a destination is unavailable.

## Success Criteria
- [ ] Search input accepts keywords like "Kyoto", "Onsen", or "Nature".
- [ ] Results card displays specific suggestions for the keyword.
- [ ] A broader "Alternative" destination is suggested (e.g., Kanazawa as an alternative to Kyoto).

## Detailed Flow
1. **Input**: User scrolls to the "Discovery" section.
2. **Query**: User enters a keyword (e.g., "Onsen").
3. **Trigger**: User clicks the "Sugerir" button.
4. **Result**: The app reveals a card with specific hot spring suggestions (e.g., Hakone) and a secondary alternative (e.g., Arima Onsen).
